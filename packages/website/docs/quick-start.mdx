# Quick Start

Follow these steps to quickly set up ESLint Plugin Boundaries in your project. Then you can explore more advanced configurations to fit your project's needs in the [Setup section](./setup/setup.md).


## 1. Installation

If you [have not installed them yet](./installation.mdx), install ESLint and ESLint Plugin Boundaries:

```bash
npm install eslint eslint-plugin-boundaries --save-dev
```

## 2. Register the Plugin

Once installed, enable the plugin in your `eslint.config.js` file:

<Tabs>
<TabItem value="JavaScript">

```javascript
import boundaries from "eslint-plugin-boundaries";

export default [
  {
    plugins: {
      boundaries,
    },
  }
];
```

</TabItem>
<TabItem value="TypeScript">

```typescript
import boundaries, { Config } from "eslint-plugin-boundaries";

export default [
  {
    plugins: {
      boundaries,
    },
  } satisfies Config
];
```

</TabItem>
</Tabs>

## 3. Define your Elements

The plugin needs to identify the elements in your project. To do so, define your elements using the [`boundaries/elements` setting](./setup/elements.md).

These patterns determine how each file or dependency is classified.

:::tip
Read the **[Elements](./setup/elements.md)** section for guidance on how to define the patterns for the elements in your project.
:::

<Tabs>
<TabItem value="JavaScript">

<CodeDiff language="javascript">{`
. import boundaries from "eslint-plugin-boundaries";

. export default [
.   {
.     plugins: {
.       boundaries,
.     },
+     settings: {
+       "boundaries/elements": [
+         { type: "controller", pattern: "controllers/*" },
+         { type: "model", pattern: "models/*" },
+         { type: "view", pattern: "views/*" }
+       ],
+     },
.   }
. ];
`}</CodeDiff>

</TabItem>
<TabItem value="TypeScript">

<CodeDiff language="typescript">{`
. import boundaries, { Config, Settings } from "eslint-plugin-boundaries";

. export default [
.   {
.     plugins: {
.       boundaries,
.     },
+     settings: {
+       "boundaries/elements": [
+         { type: "controller", pattern: "controllers/*" },
+         { type: "model", pattern: "models/*" },
+         { type: "view", pattern: "views/*" }
+       ],
+     } satisfies Settings,
.   } satisfies Config
. ];
`}</CodeDiff>

</TabItem>
</Tabs>

:::info
This is only a basic example of configuration. The plugin can be configured to identify elements being a file, or elements being a folder containing files. It also supports capturing path fragments to be used afterwards on each rule, etc.
:::

## 4. Apply Predefined Rules

Apply one of the [predefined rule configurations](./setup/eslint-integration.md) as a starting point by extending it in your `rules` section.

:::tip
You should use `recommended` config in already existing projects to gradually introduce the rules without too many errors at once, because it does not require all files in the project to belong to be part of a defined element. For new projects, you can use `strict` config to ensure all files belong to an element from the beginning.
:::

<Tabs>
<TabItem value="JavaScript">

<CodeDiff language="javascript">{`
. import boundaries from "eslint-plugin-boundaries";

. export default [
.   {
.     plugins: {
.       boundaries,
.     },
.     settings: {
.       "boundaries/elements": [
.         { type: "controller", pattern: "controllers/*" },
.         { type: "model", pattern: "models/*" },
.         { type: "view", pattern: "views/*" }
.       ],
.     },
+     rules: {
+       ...boundaries.configs.recommended.rules,
+     },
.   },
. ];
`}</CodeDiff>

</TabItem>
<TabItem value="TypeScript">

<CodeDiff language="typescript">{`
. import boundaries, { Config, Settings, Rules } from "eslint-plugin-boundaries";

. export default [
.   {
.     plugins: {
.       boundaries,
.     },
.     settings: {
.       "boundaries/elements": [
.         { type: "controller", pattern: "controllers/*" },
.         { type: "model", pattern: "models/*" },
.         { type: "view", pattern: "views/*" }
.       ],
.     } satisfies Settings,
+     rules: {
+       ...boundaries.configs.recommended.rules,
+     } satisfies Rules,
.   } satisfies Config
. ];
`}</CodeDiff>

</TabItem>
</Tabs>

## 5. Set Dependency Rules

Now you can define your own dependency rules by using the [`boundaries/dependencies` rule](./setup/rules.mdx).

:::tip
Read how to use **[Element Selectors](./setup/selectors.md)** to configure your rules in more detail.
:::

<Tabs>
<TabItem value="JavaScript">

<CodeDiff language="javascript">{`
. import boundaries from "eslint-plugin-boundaries";

. export default [
.  {
.    plugins: {
.      boundaries,
.    },
.    settings: {
.      "boundaries/elements": [
.        { type: "controller", pattern: "controllers/*" },
.        { type: "model", pattern: "models/*" },
.        { type: "view", pattern: "views/*" }
.      ],
.    },
.    rules: {
.      ...boundaries.configs.recommended.rules,
+      "boundaries/dependencies": [2, {
+        default: "disallow",
+        rules: [
+          {
+            from: { type: "controller" },
+            allow: { to: { type: ["model", "view"] } }
+          },
+          {
+            from: { type: "view" },
+            allow: { to: { type: "model" } }
+          },
+          {
+            from: { type: "model" },
+            disallow: { to: { type: "*" } }
+          }
+        ]
+      }]
.    },
.  },
. ];
`}</CodeDiff>

</TabItem>
<TabItem value="TypeScript">

<CodeDiff language="typescript">{`
. import boundaries, { Config, Settings, Rules } from "eslint-plugin-boundaries";

. export default [
.  {
.    plugins: {
.      boundaries,
.    },
.    settings: {
.      "boundaries/elements": [
.        { type: "controller", pattern: "controllers/*" },
.        { type: "model", pattern: "models/*" },
.        { type: "view", pattern: "views/*" }
.      ],
.    } satisfies Settings,
.    rules: {
.      ...boundaries.configs.recommended.rules,
+      "boundaries/dependencies": [2, {
+        default: "disallow",
+        rules: [
+          {
+            from: { type: "controller" },
+            allow: [{ type: "model" }, { type: "view" }]
+          },
+          {
+            from: { type: "view" },
+            allow: [{ type: "model" }]
+          },
+          {
+            from: { type: "model" },
+            disallow: [{ type: "*" }]
+          }
+        ]
+      }]
.    } satisfies Rules,
.  } satisfies Config
. ];
`}</CodeDiff>

</TabItem>
</Tabs>
